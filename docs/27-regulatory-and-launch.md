## 1. فاتورة ZATCA (Fatoora) — المرحلة 1

[`zatca.util.ts`](../apps/api/src/common/zatca/zatca.util.ts) يولّد رمز QR متوافقاً مع **المرحلة 1 (Generation)** بترميز **TLV**: خمسة حقول إلزامية (اسم البائع، الرقم الضريبي، الطابع الزمني، الإجمالي شاملاً الضريبة، قيمة الضريبة)، كل حقل `[tag][length][value]`، ثم Base64.

`GET /finance/invoices` يُرجع كل فاتورة مع حزمة ZATCA: `{ qr, hash (SHA-256), uuid }`. الرقم الضريبي (15 رقماً) يُشتقّ للعرض من السجل التجاري (يُهيَّأ من الإعدادات لاحقاً). البصمة تمهيدٌ لتجزئة **المرحلة 2 (Integration)** عند الربط الإنتاجي.

> مُثبَت بالاختبار: فكّ TLV يعيد الحقول الخمسة بقيم صحيحة.

## 2. النواة المالية الحيّة

أُضيفت قراءات المالية (صلاحية `finance:read` + `module.finance`):

| المسار | المحتوى |
|---|---|
| `GET /finance/summary` | القسط المكتتب، العمولة، **الأمانات خارج الميزانية**، الذمم، عدد الفواتير/السندات |
| `GET /finance/coa` | شجرة الحسابات (17 رقماً) — المستوى 1/2 مقفل، فصل داخل/خارج الميزانية |
| `GET /finance/invoices` | الفواتير الضريبية + حزمة ZATCA لكل فاتورة |
| `GET /finance/receivables` | الذمم المدينة مُجمّعة حسب العميل (من إشعارات المدين) |

الفصل الائتماني (On/Off‑Balance) يظهر في `summary.offBalanceTrust` = الإجمالي − العمولة (أمانات أقساط العملاء المحتفظ بها للمؤمِّن، خارج الميزانية) — التزاماً بمبدأ فصل أموال العملاء (انظر [20](./20-issuance-and-finance-core.md)).

## 3. لوحة الالتزام

موديول `compliance` (صلاحية `compliance:read` + `module.compliance`): `GET /compliance/overview` يعيد حالة امتثال العملاء (`groupBy`)، **توزيع مخاطر فحص PEP/العقوبات** (low/medium/high)، أنواع عمليات التحقّق، وأحدثها. مُثبَت بالاختبار: مستأجر بلا `module.compliance` يُرفض (403).

## 4. طبقة التكاملات التنظيمية

موديول `regulatory`: `GET /regulatory/status` يعرض الموصِّلات الحكومية وحالتها. **في التطوير كلها عبر Sandbox** ([BLUEPRINT §7‑ب](../BLUEPRINT.md): لا APIs حكومية حقيقية خارج الإنتاج داخل المملكة):

| الموصِّل | الفئة | الحالة |
|---|---|---|
| ZATCA / Fatoora | فوترة | مفعّل (المرحلة 1) |
| نفاذ · يقين · واثق · العنوان الوطني · فحص PEP | توثيق/هوية/سجل/عنوان/التزام | مفعّل (Sandbox) |
| نجم · مجلس الضمان الصحي/نِفيس · هيئة التأمين | مركبات/طبي/تنظيمي | مُخطّط (ربط إنتاجي لاحقاً) |

تظهر في صفحة **الإعدادات ← التكاملات** مع وسم بيئة Sandbox وتنبيه توطين البيانات.

## 5. النشر داخل المملكة

بنية تحتية تفرض القواعد غير القابلة للتفاوض (CLAUDE.md §3):

- **[`infra/k8s/`](../infra/k8s)**: `namespace` (موسومة in-kingdom)، `api.yaml` (3 نسخ + HPA 3→10 + فحوص `/health` + تقوية: non-root/readOnlyRootFS/إسقاط capabilities)، `web.yaml`، `ingress` (TLS عبر cert-manager)، `configmap` + `secrets.example` (قالب فقط)، `kustomization`.
- **[`infra/terraform/`](../infra/terraform)**: هيكل حياديّ سحابياً (AWS/GCP/Alibaba) + **حارس توطين** (`precondition` يفشل إن كانت منطقة الإنتاج خارج المملكة) + خريطة وحدات (شبكة/عنقود/قاعدة/كاش/تخزين/أسرار/سجلّات) — كلها تفرض التشفير at-rest/in-transit والنسخ الاحتياطية الموطّنة.
