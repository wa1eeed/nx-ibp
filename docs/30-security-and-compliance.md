# 30 — الأمن السيبراني والامتثال (Security & Compliance — NCA / هيئة التأمين / PDPL)

> خريطة امتثال صادقة: الضوابط **المطبَّقة فعلًا**، والفجوات، وخطة المعالجة، للوصول إلى جاهزية **الربط الحكومي** في السعودية. **هذه ليست شهادة امتثال** — بل تقييمٌ ومسارُ تحصين. الحالة الأساس مُتحقَّقة من الكود (e2e 253/253).

## 1. الأطر المرجعية في النطاق
| الإطار | الجهة | الصلة |
|---|---|---|
| **ECC** الضوابط الأساسية للأمن السيبراني | NCA | الإلزامي للجهات/المنصات؛ شرط الربط الحكومي |
| **CCC** ضوابط الأمن السيبراني للحوسبة السحابية | NCA | الاستضافة السحابية (مزوّد + مستفيد) |
| **DCC** ضوابط أمن البيانات | NCA | تصنيف البيانات وحمايتها وتوطينها |
| اشتراطات الأمن وحوكمة تقنية المعلومات | **هيئة التأمين** | قطاع التأمين (توطين، تدقيق، إبلاغ الحوادث) |
| **PDPL** نظام حماية البيانات الشخصية | SDAIA | البيانات الشخصية والموافقة والتوطين |

> **مهم:** ECC ليس كودًا فقط — بل **حوكمة + تقنية + تشغيل** (سياسات، إدارة مخاطر، توعية، تدقيق). الكود يغطّي الجانب التقني؛ البقية برنامج مؤسسي.

## 2. الضوابط المطبَّقة فعلًا (مخرَّطة على مجالات ECC)
| مجال ECC | الضابط | الحالة |
|---|---|---|
| الدفاع — الهوية والدخول (IAM) | JWT + حارس عالمي · RBAC + Entitlements مزدوج · bcrypt · أقل امتياز | ✅ |
| الدفاع — حماية البيانات | تشفير at-rest (AES-256-GCM لاعتماد ZATCA) · TLS in-transit · لا أسرار في الكود (env) | ✅ جزئي |
| الدفاع — عزل المستأجرين | عزل ثلاث طبقات (FK + سياق + Prisma middleware) — مُثبَت بالاختبار | ✅ |
| الدفاع — تقوية الحدود | Helmet (رؤوس) · CORS مضبوط · ValidationPipe · حاويات non-root/readOnlyRootFS | ✅ |
| الدفاع — التشفير | خوارزميات معيارية (AES-256-GCM · SHA-256 · bcrypt · ECDSA secp256k1) | ✅ |
| الدفاع — السجلّات | `auditLog` لكل عملية حسّاسة (تطبيقي) | ✅ جزئي |
| الملفات | روابط موقّتة فقط · allowlist (SVG محظور) · عزل بالمسار + UUID | ✅ |
| الصمود — التوفّر | k8s نسخ متعددة + HPA + فحوص صحّة | ✅ (بنية) |
| الطرف الثالث/السحابة | تصميم حياديّ سحابياً + **حارس توطين** (Terraform يمنع إنتاجًا خارج المملكة) | ✅ (بنية) |

## 4. التوطين والاستضافة (الوضع والخطة)
- **الوضع الحالي (مؤقّت ومُقَر):** الاستضافة على VPS **خارج المملكة** — **غير مسموح لبيانات الإنتاج الحقيقية** بموجب PDPL/هيئة التأمين، لكنه مقبول مؤقتًا لبيانات وهمية/تجريبية وما قبل الإطلاق الفعلي.
- **شرط الربط الحكومي:** أي تكامل حكومي (هيئة التأمين/ZATCA إنتاجي/نفاذ) يتطلّب **استضافة داخل المملكة** + استيفاء ECC/CCC + اتفاقيات. لا يُفعَّل الربط الحقيقي قبل ذلك (حاليًا Sandbox فقط).
- **خطة النقل:** المعمار حياديّ سحابياً؛ النقل لمزوّد داخل المملكة (AWS me-central-1 / GCP / Alibaba / محلي) = تغيير `.env` + تجسيد Terraform، **دون تغيير كود**. حارس التوطين في Terraform يمنع نشر الإنتاج خارج المملكة آليًا.
- **توطين شامل:** القاعدة + النسخ الاحتياطية + السجلّات + المرفقات داخل المملكة (يشمل اختيار وجهة تخزين الكائنات داخل المملكة بدل R2 للإنتاج).

## 5. مسار المعالجة (Track G — تحصين أمني)
يُنفَّذ على دفعات قبل أي ربط حكومي/إطلاق حقيقي:
1. **P0 — صلابة الدخول:** ✅ **مكتمل** — قفل الحساب ضد القوّة الغاشمة (Redis، 8 محاولات/15 دقيقة، تصفير عند النجاح، عبر `RateLimitService` على كل بوّابات الدخول الثلاث: موظف/منصّة/عميل) + **سياسة كلمات المرور** (8 أحرف، حرف كبير/صغير/رقم) + **MFA (TOTP) لسوبر أدمن المنصة** (setup/enable/فرض عند الدخول + `/admin/security`). متبقٍّ: توسعة MFA لموظفي الشركات. اختبارات `security.e2e` + `mfa.e2e`.
2. **P1 — الجلسات والأسرار:** refresh tokens + تدوير · نقل الأسرار إلى KMS/Vault.
3. **P1 — البيانات:** تصنيف + تشفير حقول PII + سياسة احتفاظ/حذف (PDPL).
4. **P1 — الرصد:** سجلّات مركزية + تنبيهات + WAF + لوحة مراقبة.
5. **P1 — SDLC آمن:** SCA/SAST/DAST في CI + إدارة ثغرات + ترقيع.
6. **P1 — الصمود:** نسخ احتياطي مؤتمت + اختبار DR + خطة استجابة حوادث + الإبلاغ.
7. **حوكمة:** توثيق السياسات (مخاطر/تصنيف/توعية/أدوار) لاستيفاء ECC الإداري.
8. **تدقيق اختراق** مستقل قبل الإنتاج.

## 5.5 متطلبات الإسناد (Outsourcing) واتفاقية مستوى الخدمة (SLA) — لائحة هيئة التأمين
البنود الهندسية الواجب توفّرها في العقد الفني بين المنصة وعملائها (شركات الوساطة/التأمين)، ومطابقة NCA ECC/CCC. **الحالة بصدق:**

| # | المتطلب | الحالة | ملاحظة |
|---|---|---|---|
| 1 | **تشفير النقل** (TLS 1.3) | ✅ (بنية) | عبر عاكس Coolify/Ingress؛ يُفرض TLS حديث |
| 2 | **تشفير التخزين** (AES-256) | ~ جزئي | بيانات ZATCA مشفّرة AES-256-GCM؛ **تشفير القاعدة الكامل at-rest = قاعدة مُدارة داخل المملكة (بنية)** |
| 3 | **عزل بيئة البيانات** (Multi-tenant) | ✅ | 3 طبقات (FK + ALS + Prisma middleware)، مُثبَت باختبار العزل |
| 4 | **سياسة الاحتفاظ والإتلاف الآمن** (Data Sanitization + سجلّ إتلاف) | ❌ **فجوة** | يلزم ميزة مسح نهائي لبيانات العميل عند انتهاء العقد + سجلّ يثبت الإتلاف |
| 5 | **إتاحة 99.9% (Uptime)** | ~ تصميم | k8s نسخ متعددة + HPA + فحوص صحّة؛ يُفعَّل عند النشر داخل المملكة |
| 6 | **RPO دقائق (نسخ لحظي)** + **RTO ≤ ساعتين (Failover آلي)** | ❌ **فجوة** | يلزم تجسيد النسخ الاحتياطي اللحظي + خطة تعافٍ آلية واختبارها |
| 7 | **حق التفتيش والوصول (Right to Audit)** | ~ جزئي | سجلّات التدقيق متاحة؛ يلزم **بند تعاقدي صريح** + تمكين مراجعة المفتّشين |
| 8 | **السيادة الوطنية للبيانات** (استضافة داخل المملكة، مركز فئة C+ من CST) | ⏳ مخطّط | حارس توطين في Terraform يمنع الإنتاج خارج المملكة؛ الاستضافة الحالية مؤقّتة لبيانات وهمية |
| 9 | **منع تسريب البيانات (DLP)** | ❌ **فجوة** | يلزم ضوابط منع نسخ/تصدير غير مصرّح + قيود على التصدير |
| 10 | **المصادقة الثنائية (MFA) إلزامية** | ~ جزئي | ✅ **منفّذة لسوبر أدمن المنصة** (TOTP، setup/enable/فرض عند الدخول + صفحة `/admin/security`)؛ متبقٍّ: توسعتها لموظفي الشركات |
| 11 | **مبدأ الامتيازات الأقل (RBAC دقيق)** | ✅ | مصفوفة CRUD لكل موظف/موديول + فحص مزدوج |
| 12 | **سجلّات غير قابلة للتعديل (Immutable) مشفّرة، احتفاظ ≥ سنة** | ✅ (تطبيق) + ~ (بنية) | ✅ **`AuditLog` غير قابل للتعديل/الحذف** (يُرفض update/delete/upsert في Prisma حتى من الكود) + **يلتقط IP/الجهاز/المستخدم/الإجراء/الوقت** + **مراجعة/تصدير للمفتّشين** (`GET /platform/audit`). متبقٍّ (بنية): خادم سجلّات منفصل + تشفير القرص + فرض احتفاظ سنة. اختبار `audit-immutable.e2e` |
| 13 | **اختبار اختراق سنوي** (Black/Grey box، جهة مرخّصة NCA) | ❌ عملية | قبل الإنتاج + دوريًا (SQLi/IDOR) |
| 14 | **إدارة الثغرات وسدّها** (Critical/High خلال 48–72 س + إشعار العميل) | ❌ عملية | يلزم SCA/SAST في CI + سياسة ترقيع + إشعار |

### RBAC — لا صلاحيات مطلقة (NCA ECC)
- **الصلاحيات مقسّمة برمجيًا** (module × CRUD) لكل دور، **قابلة للتخصيص لكل موظف** — فيمكن إنشاء دور «مسؤول نظام» يقتصر على `settings`/إدارة المستخدمين **دون** `finance`/`clients`/`production` (لا يرى الوثائق أو البيانات المالية الحسّاسة)، ودور «التزام» بصلاحية قراءة فقط. **دور «مالك الحساب»** الافتراضي كامل الصلاحيات (للتأسيس)، لكن الشركة تُنشئ أدوارًا مقيّدة وفق فصل المهام.
- **لا تصدير لقاعدة البيانات كاملة**؛ كل استعلام مفلتر بالمستأجر + RBAC.
- **فجوة (تعزيز):** حصر رؤية الموظف على **محفظته** فقط (owner-based row-level) — حاليًا العزل بالمستأجر لا بالموظف؛ يُضاف عند الحاجة.

> **الخلاصة:** مطبَّق فعلاً: العزل، RBAC الدقيق، **سجلّ تدقيق ثابت (immutable) يلتقط IP/الجهاز**، **قفل قوّة غاشمة + سياسة كلمات + MFA (TOTP) لأدمن المنصة**، **فصل المهام في الاعتماد المالي**، تشفير الأسرار، تصميم التوطين والإتاحة. **الفجوات التي تلزم للإطلاق التعاقدي:** الاحتفاظ/الإتلاف الآمن، RPO/RTO المُشغَّل، DLP، توسعة MFA للموظفين، فصل/تشفير خادم السجلّات + احتفاظ سنة، اختبار الاختراق، إدارة الثغرات، والاستضافة داخل المملكة. تُدرَج في [29 — خطّة العمل](./29-roadmap-next.md) ضمن Track G/الامتثال.

## 6. الخلاصة
أساس تقني **قويّ** (عزل + IAM + تشفير + تقوية + تدقيق + تصميم توطين). للوصول لجاهزية **NCA ECC/CCC + هيئة التأمين + الربط الحكومي + العقد الفني (SLA/الإسناد)** يلزم استكمال **Track G + §5.5** + **النقل داخل المملكة** + **الجانب الحوكمي**. لا يُفعَّل أي ربط حكومي حقيقي قبل استيفاء ذلك (التكامل الحالي Sandbox).
