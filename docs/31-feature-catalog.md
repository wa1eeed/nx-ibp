# 31 — كتالوج المزايا والقدرات (Feature & Capabilities Catalog)

> ملخّص تنفيذي لقدرات منصّة IBP — موجّه للأعمال والمشترين (Due-Diligence). كل ما هنا **منفَّذ ومُختبَر** (e2e 323/323)؛ القادم في [29](./29-roadmap-next.md). التموضع: **InsurTech SaaS · نظام تشغيل وساطة التأمين**.

## 1. المنصّة بإيجاز
- **SaaS متعدد المستأجرين** لشركات وساطة التأمين في السعودية، يؤتمت دورة العمل كاملةً ويُباع باشتراكات بباقات وصلاحيات.
- **4 واجهات**: لاندينق بيج تسويقي · لوحة موظف/أدمن الشركة · لوحة السوبر أدمن (مالك المنصّة) · بوّابة العميل.
- **ثنائية اللغة (عربي/إنجليزي) + RTL** بالكامل.

## 2. القدرات حسب المجال
| المجال | القدرات |
|---|---|
| **تعدّد المستأجرين والعزل** | عزل بيانات ثلاثي الطبقات لكل شركة · نطاقات منفصلة (منصّة/مستأجر/عميل) · تخزين معزول بالمسار |
| **الباقات والاشتراكات** | باقات (Basic/Premium/Enterprise) · **تسعير لكل مستخدم بلا سقف** (شهري/سنوي + نسبة توفير + تجربة مجانية) · **زمن استجابة دعم (SLA) لكل باقة** (قابل للتهيئة) · Entitlements لكل موديول + مفاتيح `feature.*` للقدرات · إضافات (add-ons) · **هيكلة وفق هيئة التأمين** · **السوبر أدمن يحرّر السعر/التجربة/SLA ويفعّل/يعطّل الميزات لكل باقة** (ينعكس فورًا في اللاندينق والمقارنة) |
| **التسجيل والـOnboarding والمقارنة** | معالج تسجيل ذاتي بتأطير **«ادفع بقدر نموّك»** (باقة → عدد مستخدمين → سعر حيّ بإجمالي مشطوب + شارة تجربة + تفصيل شفّاف) · **onboarding** بحقول مُتحقَّقة (رقم موحّد 10 · ضريبي 15 · جوال `05XXXXXXXX`) · **صفحة/قسم مقارنة الباقات** (تشمل SLA) · **مسار «تواصل معنا» (Lead)** · **تجاوب موبايل كامل** |
| **الصلاحيات (RBAC)** | بُعدان: باقة الشركة + دور الموظف · أدوار قابلة للتخصيص بمصفوفة CRUD+Revert لكل موديول (13 موديولاً) · **12 قالب دور جاهز + هيكل تنظيمي افتراضي (7 أقسام) يُزوَّدان تلقائيًا لكل حساب جديد** · **صلاحيات على مستوى المنتج (فرع التأمين)**: حصر رؤية/إنشاء الموظف بفروع محدّدة (افتراضي = كل الفروع، متوافق رجعيًا) |
| **المنتجات والكتالوج** | كتالوج فروع التأمين (10 فئات/47 فرعًا) بالمعالجة الضريبية لكل فئة + جاهزية النموذج + **إحصاءات إنتاج المستأجر لكل فرع** + طلب سريع بالفرع |
| **المستندات المركزية** | مستودع موحّد لكل مستندات الشركة (رسمية/مرفقات) بفلترة/بحث وروابط للكيان المرتبط (عميل/وثيقة/مطالبة) + روابط عرض موقّتة |
| **العملاء والتأهيل** | منشآت/أفراد · تفرّد · بوّابة الالتزام (اعتماد/رفض) · بحث وتبويبات · **حقول مُتحقَّقة**: السجل التجاري/الهوية (10 أرقام) · الرقم الضريبي (15 رقمًا) · **جوال سعودي** (05) + **هاتف ثابت** (01) + **اسم شخص التواصل** |
| **التحقّق الحكومي (KYC/KYB)** | يقين · واثق · نفاذ · العنوان الوطني · فحص PEP/العقوبات · تعبئة ذكية · أرصدة عمليات لكل مزوّد |
| **النموذج الديناميكي** | محرّك مدفوع بمخطّط (DSL) لكل فرع تأمين · تحقّق مدخلات صارم · كتل متكرّرة (مركبات/أعضاء/مواقع) · **مكتبة قوالب قابلة لإعادة الاستخدام** (تعبئة مسبقة + «حفظ كقالب» + عدّاد استخدام) · **استيراد جماعي من CSV** للكتل المتكررة (أساطيل/مجموعات): تنزيل قالب + رفع ملف + تحقّق كل صفّ + تقرير أخطاء دقيق |
| **الاكتتاب وعروض الأسعار** | طلب عروض (RFQ/Slip) · عروض هجينة · **جدول مقارنة آلي يحدّد الأنسب** · أمر إسناد (Firm Order) · **نموذج عرض بحساب حيّ**: الضريبة/الإجماليات/العمولة تلقائيًا بنسبة الفرع (حياة 0%) بلا إدخال يدوي · **بحث سريع عن المنتج** في إنشاء الطلب |
| **الإنتاج والإصدار** | إصدار من العرض المختار · **سلسلة اعتماد ديناميكية قابلة للتهيئة** (بوّابة فنية تُفعَّل/تُعطَّل + خطوات إضافية بدور لكلٍّ) مع **حارس امتثالي: فصل المهام** (المعتمِد المالي ≠ المُصدِر) · مولّد تسلسلي بالفرع · **ملاحق احترافية (Endorsements)**: اتجاه مالي (قسط إضافي/مُرتجَع) + معاينة ضريبة حيّة بنسبة الفرع + توليد إشعار مدين/دائن تلقائيًا |
| **المالية والمحاسبة** | **قيد JRV مزدوج متوازن** · شجرة حسابات 17 رقماً (مقفلة المستوى 1/2) · فصل ائتماني On/Off-Balance (أمانات العملاء) · **الضريبة حسب فرع التأمين** (الحياة معفى 0%؛ البقية 15%) · **ضريبة العمولة/الرسوم كالتزام Output VAT** · **دورة تحصيل كاملة** (سند قبض RCV + ذمم حيّة + كشف حساب) · **الإلغاء** (قسط مُرتجَع نسبةً + إشعار دائن CNP/CNC) · **تسوية المؤمِّن** (PYV + أعمار دَين) + **ميزان مراجعة متوازن** · **نظرة المالك**: قائمة دخل مبسّطة (عمولات + رسوم − عمولات الوسطاء الفرعيين = صافي الدخل، الضريبة مستبعَدة كتحصيل نيابة) + مؤشّرات صحة الأعمال (القسط المُدار، معدّل العمولة الفعلي، تحصيل العمولات، الذمم، الأمانات، نسبة الخسارة) + اتجاه صافي الدخل 6 أشهر · **نموذج تحصيل قابل للتهيئة** (تحصيل كامل القسط بأمانات ↔ العميل يدفع للمؤمِّن مباشرةً والوسيط يُفوتر عمولته فقط) — يُضبط على العميل ويُبصَم على الوثيقة عند الإصدار (سلامة تاريخية) |
| **اتجاه الفوترة (ZATCA)** | فاتورة **العمولة على المؤمِّن** · فاتورة **رسوم الخدمة على العميل** (إيراد مستقلّ 04020) · فصل صحيح يظهر «الطرف» ولا تُكشف عمولة الوسيط للعميل · **حقول الفاتورة الإلزامية**: رمز نوع الفاتورة (388) + بنود مفصّلة بفئة/نسبة ضريبة لكل بند + الرقم الضريبي والعنوان الوطني للبائع والمشتري |
| **الوسطاء الفرعيون** | سجلّ وسطاء فرعيين مرخّصين (ترخيص الهيئة/نسبة عمولة) · **احتساب آلي لحصّتهم** من العمولة · دفتر مستحقّ/مُسوّى/متبقّي لكل وسيط فرعي · **صرف عمولاتهم** (سند PYV بقيد مصروف) |
| **إدارة شركات التأمين** | كيان «شركة تأمين» كامل: سجلّ المؤمِّنين + **نِسب العمولة/الاتفاقيات** + **دورة التسوية (مهلة السداد)** + **الحساب البنكي (بنك/آيبان)** + الترخيص/التواصل · **إحصاءات إنتاج فعلية لكل شركة** (عدد وثائق/أقساط/عمولة) |
| **البريد متعدّد المستأجرين (BYO Resend)** | كل شركة ترسل من **نطاقها الرسمي** عبر حساب Resend خاص (ربط آلي + DNS + تحقّق) · **مفتاح مشفّر at-rest** (لا يُكشف) · **fallback مركزي بلا انقطاع** + **ترقية آلية** بعد التحقّق · **وضع الردود فقط** (Reply-To بريد الشركة بلا Resend) · قوالب HTML **ثنائية اللغة** بهوية المستأجر (34 نوعًا) |
| **الهوية البصرية (White-label)** | لون أساسي + شعار (رابط عام ثابت يظهر بالبريد) + اسم عرض — ينعكس **ديناميكيًا** على واجهة كل الموظفين + **بوّابة العميل** + **البريد** + **الوثائق المطبوعة** · معاينة حيّة · عزل تام |
| **الأهداف والأداء (Targets)** | أهداف إنتاج للوسطاء الفرعيين/فروع التأمين (أقساط/عدد وثائق/عمولات · شهري/ربعي/سنوي) · **الفعلي محسوب من الإنتاج القائم** + % الإنجاز بأشرطة تقدّم + فلترة الفترة |
| **الإشعارات (لكامل النظام)** | إشعارات **العملاء** (Email/SMS) و**الموظفين** (توجيه حسب الوحدة + مالك الحساب) · 34 نوعًا **ثنائي اللغة** بمستويين (منصّة/شركة) قابلة للتفعيل/التعطيل وتعديل النص · **مركز إشعارات داخل المنصة (جرس in-app) + النقر ينقل لمكان العمل** (مهمة CRM ⇒ المهام، مطالبة ⇒ المطالبات…) · البريد عبر `sendTenantEmail` (هوية/نطاق المستأجر) |
| **ZATCA Fatoora (المرحلة 2)** | تهيئة معزولة لكل مستأجر (CSR/CCSID/PCSID مشفّرة) · خطّ تهيئة 4 خطوات · مستند فوترة موحّد (UUID + عدّاد/تجزئة معزولة + QR TLV + UBL) · توجيه B2B مقاصة / B2C إبلاغ |
| **الخدمة والمطالبات** | طلبات خدمة (إضافة/حذف/تعديل/استفسار) بدورة حالات **+ إسناد لموظف + أولوية (عاجل/عالٍ/عادي/منخفض) + ملاحظات وخطّ زمني + فلاتر (الحالة/طلباتي) + مؤشّر عمر الطلب** · مطالبات بدورة كاملة · تجديدات (الوثائق المستحقّة) |
| **سجلّ الشكاوى (امتثال)** | متطلب هيئة التأمين: تسجيل الشكاوى بفئة/مصدر/أولوية + **مهلة معالجة (SLA)** + إسناد + **تصعيد للهيئة** + ملخّص معالجة وخطّ زمني · **تقرير تنظيمي** (تجميع بالفئة/الحالة + التزام SLA% + متوسّط زمن المعالجة + المُصعَّدة) — بصلاحية الالتزام |
| **التقارير والتحليلات** | لوحة مؤشّرات حيّة · عمولات · إنتاج (GWP/تحويل/حسب الفرع والشركة) · مطالبات (نسبة الخسارة) · تقرير هيئة التأمين · كتالوج 12 تقريراً |
| **المستندات** | رفع/عرض عبر روابط موقّتة فقط · عزل بالمسار · حدّ الباقة · allowlist (SVG محظور) · تمييز رسمي/مرفق · **وثيقة فاتورة ضريبية مطبوعة/PDF بهوية المستأجر** (HTML→طباعة، RTL دقيق + ZATCA) |
| **التملّك (Ownership) — للمؤسسات** | صفحة مستقلّة (بيع استشاري): **رخصة Source-Available** أو **نقل ملكية فكرية كاملة** + نشر مرن (On-premise/سحابة خاصة) + دعم اختياري + 4 ركائز + نموذج تواصل |
| **بوّابة العميل** | وثائقه · مطالباته · طلباته · كشف حسابه (+ جدول الأقساط) · مستنداته (عزل مزدوج) · **توفير الدخول**: الموظف يدعو العميل ببريد ⇒ رابط تفعيل، والعميل **يعيّن كلمة مروره ذاتيًا** ثم دخول تلقائي (لا يضبط الموظف كلمة المرور) + إلغاء الدخول · **الدفع الإلكتروني**: يدفع أقساطه/ذممه أونلاين ⇒ سند قبض تلقائي |
| **بوّابة الدفع (BYO)** | شركة الوساطة تفعّل **بوّابة واحدة** (Tap/Moyasar) بمفاتيحها (**السرّي مشفّر at-rest**) لاستقبال مدفوعات عملائها؛ العميل يدفع من كشف الحساب ⇒ شحنة عبر بوّابة المستأجر ⇒ عودة/**webhook موقّع** ⇒ **سند قبض تلقائي** (شلال الأقساط) بحتمية. اشتراكات المنصّة عبر Tap (تجريد بوّابة موحّد) |
| **الهيكل الإداري والأقسام** | هيكل هرمي بمخطّط شجري (RTL) · قسم بدور افتراضي يُورَّث للموظف المُسنَد · **الأقسام السبعة + أدوارها مُزوَّدة تلقائيًا عند التسجيل** (الإدارة العليا + مبيعات/التزام/اكتتاب/مالية/خدمة/مطالبات) |
| **السوبر أدمن** | إدارة الحسابات (تعليق/تفعيل) · **تحرير أسعار الباقات (لكل مستخدم شهري/سنوي) + أيام التجربة + تفعيل/تعطيل الميزات لكل باقة** · متابعة طلبات «تواصل معنا» (Leads) · الاستخدام الكلّي |
| **الأمان والتدقيق** | JWT + bcrypt · **قفل قوّة غاشمة (Redis) + سياسة كلمات + MFA لسوبر أدمن المنصة (TOTP)** · Helmet/CORS/تحقّق · تشفير at-rest (AES-256-GCM) · **سجل تدقيق ثابت (immutable) يلتقط IP/الجهاز** لكل عملية حسّاسة (انظر [30](./30-security-and-compliance.md)) |
| **حياد السحابة والنشر** | طبقة تخزين حيادية · k8s + Terraform (حارس توطين داخل المملكة) · Docker |

## 3. الحوكمة والامتثال
- **عزل مُثبَت بالاختبار** ثنائي/ثلاثي الاتجاه (مستأجر/منصّة/عميل).
- **تصميم توطين** داخل المملكة (PDPL/NCA) + حارس Terraform يمنع إنتاجًا خارجها.
- خريطة امتثال **NCA ECC / هيئة التأمين / PDPL** وخطة التحصين في [30](./30-security-and-compliance.md).
- بيئة تطوير ببيانات وهمية فقط؛ التكامل الحكومي عبر Sandbox حتى استيفاء شروط الإنتاج.

## 4. الجاهزية التقنية
- **323/323 اختبار e2e** عبر 45 ملفاً (عزل، صلاحيات + تزويد الأقسام/الأدوار عند التسجيل، تسعير لكل مستخدم + onboarding + Lead، **البريد BYO + الهوية البصرية + رفع الشعار**، **الأهداف والأداء**، **نظرة المالك المالية (قائمة دخل متّسقة)**، **وثيقة الفاتورة المطبوعة + حقول ZATCA الإلزامية**، **تطوير خدمة العملاء (إسناد/أولوية/خطّ زمني)**، حسابات مالية + دورة تحصيل/تسوية/ميزان مراجعة + **القيود اليدوية/المصروفات + عمولات الموظفين + خطة تقسيط الأقساط (شلال تحصيل) + القوائم المالية الثلاث: الميزانية العمومية + دفتر الأستاذ + إقرار ض.ق.م + بيان التدفّق النقدي**، اتجاه الفاتورة، الملاحق، الوسطاء الفرعيون، كشف المؤمِّن الدوري، **مكافحة غسل الأموال (تقييم مخاطر + فرز عقوبات/PEP + بلاغات اشتباه STR)**، قوالب النماذج، ضريبة/فرع، ZATCA، إشعارات وتوجيهها، سلاسل الاعتماد وفصل المهام، سجل ثابت، MFA، بوّابة).
- TypeScript صارم · تحقّق مدخلات على الحدود · معالجة أخطاء موحّدة · توثيق شامل (35 وثيقة).
- قاعدة اختبار منفصلة (`ibp_test`) لا تلوّث بيانات العرض.
