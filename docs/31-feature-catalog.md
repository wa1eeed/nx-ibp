# 31 — كتالوج المزايا والقدرات (Feature & Capabilities Catalog)

> ملخّص تنفيذي لقدرات منصّة IBP — موجّه للأعمال والمشترين (Due-Diligence). كل ما هنا **منفَّذ ومُختبَر** (e2e 241/241)؛ القادم في [29](./29-roadmap-next.md). التموضع: **InsurTech SaaS · نظام تشغيل وساطة التأمين**.

## 1. المنصّة بإيجاز
- **SaaS متعدد المستأجرين** لشركات وساطة التأمين في السعودية، يؤتمت دورة العمل كاملةً ويُباع باشتراكات بباقات وصلاحيات.
- **4 واجهات**: لاندينق بيج تسويقي · لوحة موظف/أدمن الشركة · لوحة السوبر أدمن (مالك المنصّة) · بوّابة العميل.
- **ثنائية اللغة (عربي/إنجليزي) + RTL** بالكامل.

## 2. القدرات حسب المجال
| المجال | القدرات |
|---|---|
| **تعدّد المستأجرين والعزل** | عزل بيانات ثلاثي الطبقات لكل شركة · نطاقات منفصلة (منصّة/مستأجر/عميل) · تخزين معزول بالمسار |
| **الباقات والاشتراكات** | باقات (Basic/Premium/Enterprise) · **تسعير لكل مستخدم بلا سقف** (شهري/سنوي + نسبة توفير + تجربة مجانية) · Entitlements لكل موديول + مفاتيح `feature.*` للقدرات · إضافات (add-ons) · **هيكلة وفق هيئة التأمين** (الأساسية = كل الجوهر التشغيلي/الامتثالي) · **السوبر أدمن يحرّر السعر/أيام التجربة ويفعّل/يعطّل الميزات لكل باقة** (ينعكس فورًا في اللاندينق والمقارنة) |
| **التسجيل والـOnboarding والمقارنة** | معالج تسجيل ذاتي (اختيار باقة → عدد مستخدمين → سعر حيّ بإجمالي مشطوب + شارة تجربة) · **onboarding** بحقول مُتحقَّقة (رقم موحّد 10 · ضريبي 15 · جوال `05XXXXXXXX`) · **صفحة/قسم مقارنة الباقات** بقيم وصفية (مشمول/إضافة/حسب الاستخدام/حصّة) · **مسار «تواصل معنا» (Lead)** للمؤسسات · **تجاوب موبايل كامل** (درج تنقّل) |
| **الصلاحيات (RBAC)** | بُعدان: باقة الشركة + دور الموظف · أدوار قابلة للتخصيص بمصفوفة CRUD+Revert لكل موديول (13 موديولاً) · **12 قالب دور جاهز + هيكل تنظيمي افتراضي (7 أقسام) يُزوَّدان تلقائيًا لكل حساب جديد** |
| **العملاء والتأهيل** | منشآت/أفراد · تفرّد · بوّابة الالتزام (اعتماد/رفض) · بحث وتبويبات |
| **التحقّق الحكومي (KYC/KYB)** | يقين · واثق · نفاذ · العنوان الوطني · فحص PEP/العقوبات · تعبئة ذكية · أرصدة عمليات لكل مزوّد |
| **النموذج الديناميكي** | محرّك مدفوع بمخطّط (DSL) لكل فرع تأمين · تحقّق مدخلات صارم · كتل متكرّرة (مركبات/أعضاء/مواقع) · **مكتبة قوالب قابلة لإعادة الاستخدام** (تعبئة مسبقة + «حفظ كقالب» + عدّاد استخدام) |
| **الاكتتاب وعروض الأسعار** | طلب عروض (RFQ/Slip) · عروض هجينة · **جدول مقارنة آلي يحدّد الأنسب** · أمر إسناد (Firm Order) |
| **الإنتاج والإصدار** | إصدار من العرض المختار · **سلسلة اعتماد ديناميكية قابلة للتهيئة** (بوّابة فنية تُفعَّل/تُعطَّل + خطوات إضافية بدور لكلٍّ) مع **حارس امتثالي: فصل المهام** (المعتمِد المالي ≠ المُصدِر) · مولّد تسلسلي بالفرع |
| **المالية والمحاسبة** | **قيد JRV مزدوج متوازن** · شجرة حسابات 17 رقماً (مقفلة المستوى 1/2) · فصل ائتماني On/Off-Balance (أمانات العملاء) · **الضريبة حسب فرع التأمين** (الحياة معفى 0%؛ البقية 15%) · **ضريبة العمولة/الرسوم كالتزام Output VAT** · **دورة تحصيل كاملة** (سند قبض RCV + ذمم حيّة + كشف حساب) · **الإلغاء** (قسط مُرتجَع نسبةً + إشعار دائن CNP/CNC) · **تسوية المؤمِّن** (PYV + أعمار دَين) + **ميزان مراجعة متوازن** |
| **اتجاه الفوترة (ZATCA)** | فاتورة **العمولة على المؤمِّن** · فاتورة **رسوم الخدمة على العميل** (إيراد مستقلّ 04020) · فصل صحيح يظهر «الطرف» ولا تُكشف عمولة الوسيط للعميل |
| **المنتِجون (الوسطاء الفرعيون)** | سجلّ منتِجين مرخّصين (ترخيص الهيئة/نسبة عمولة) · **احتساب آلي لحصّتهم** من العمولة · دفتر مستحقّ/مُسوّى/متبقّي لكل منتِج · **صرف عمولاتهم** (سند PYV بقيد مصروف) |
| **الإشعارات (لكامل النظام)** | إشعارات **العملاء** (Email/SMS) و**الموظفين** (توجيه حسب الوحدة + مالك الحساب) · 18 نوعًا بمستويين (منصّة/شركة) قابلة للتفعيل/التعطيل وتعديل النص · **مركز إشعارات داخل المنصة (جرس in-app)** للموظف والعميل · بوّابة seam جاهزة (Taqnyat/Resend) |
| **ZATCA Fatoora (المرحلة 2)** | تهيئة معزولة لكل مستأجر (CSR/CCSID/PCSID مشفّرة) · خطّ تهيئة 4 خطوات · مستند فوترة موحّد (UUID + عدّاد/تجزئة معزولة + QR TLV + UBL) · توجيه B2B مقاصة / B2C إبلاغ |
| **الخدمة والمطالبات** | طلبات خدمة (إضافة/حذف/تعديل/استفسار) بدورة حالات · مطالبات بدورة كاملة · تجديدات (الوثائق المستحقّة) |
| **التقارير والتحليلات** | لوحة مؤشّرات حيّة · عمولات · إنتاج (GWP/تحويل/حسب الفرع والشركة) · مطالبات (نسبة الخسارة) · تقرير هيئة التأمين · كتالوج 12 تقريراً |
| **المستندات** | رفع/عرض عبر روابط موقّتة فقط · عزل بالمسار · حدّ الباقة · allowlist (SVG محظور) · تمييز رسمي/مرفق |
| **بوّابة العميل** | وثائقه · مطالباته · طلباته · كشف حسابه · مستنداته (عرض فقط، عزل مزدوج) |
| **الهيكل الإداري والأقسام** | هيكل هرمي بمخطّط شجري (RTL) · قسم بدور افتراضي يُورَّث للموظف المُسنَد · **الأقسام السبعة + أدوارها مُزوَّدة تلقائيًا عند التسجيل** (الإدارة العليا + مبيعات/التزام/اكتتاب/مالية/خدمة/مطالبات) |
| **السوبر أدمن** | إدارة الحسابات (تعليق/تفعيل) · **تحرير أسعار الباقات (لكل مستخدم شهري/سنوي) + أيام التجربة + تفعيل/تعطيل الميزات لكل باقة** · متابعة طلبات «تواصل معنا» (Leads) · الاستخدام الكلّي |
| **الأمان والتدقيق** | JWT + bcrypt · **قفل قوّة غاشمة (Redis) + سياسة كلمات + MFA لسوبر أدمن المنصة (TOTP)** · Helmet/CORS/تحقّق · تشفير at-rest (AES-256-GCM) · **سجل تدقيق ثابت (immutable) يلتقط IP/الجهاز** لكل عملية حسّاسة (انظر [30](./30-security-and-compliance.md)) |
| **حياد السحابة والنشر** | طبقة تخزين حيادية · k8s + Terraform (حارس توطين داخل المملكة) · Docker |

## 3. الحوكمة والامتثال
- **عزل مُثبَت بالاختبار** ثنائي/ثلاثي الاتجاه (مستأجر/منصّة/عميل).
- **تصميم توطين** داخل المملكة (PDPL/NCA) + حارس Terraform يمنع إنتاجًا خارجها.
- خريطة امتثال **NCA ECC / هيئة التأمين / PDPL** وخطة التحصين في [30](./30-security-and-compliance.md).
- بيئة تطوير ببيانات وهمية فقط؛ التكامل الحكومي عبر Sandbox حتى استيفاء شروط الإنتاج.

## 4. الجاهزية التقنية
- **241/241 اختبار e2e** عبر 36 ملفاً (عزل، صلاحيات + تزويد الأقسام/الأدوار عند التسجيل، تسعير لكل مستخدم + onboarding + Lead، حسابات مالية + دورة تحصيل/تسوية/ميزان مراجعة، اتجاه الفاتورة، المنتِجون، قوالب النماذج، ضريبة/فرع، ZATCA، إشعارات وتوجيهها، سلاسل الاعتماد وفصل المهام، سجل ثابت، MFA، بوّابة).
- TypeScript صارم · تحقّق مدخلات على الحدود · معالجة أخطاء موحّدة · توثيق شامل (35 وثيقة).
- قاعدة اختبار منفصلة (`ibp_test`) لا تلوّث بيانات العرض.
