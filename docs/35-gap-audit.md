# 35 — التدقيق الشامل للنواقص (منظور رئيس شركة وساطة تأمين سعودية)

> فحصٌ من الألف إلى الياء لكل أقسام المنصة وصفحاتها وبوّاباتها وصلاحياتها، مقابل **متطلبات هيئة التأمين السعودية + ZATCA + التشغيل الفعلي لشركات الوساطة**. الهدف: تحديد ما ينقص ليصبح النظام أفضل منصّة لإدارة نشاط الوساطة بجميع أقسامها. **القسم المالي هو الأخطر** — بدون اكتماله لا تؤدّي المنصّة دورها.
>
> **المنهج:** كل بند مُتحقَّق من الكود (✅ موجود · ⚠️ جزئي · ❌ مفقود). الأولويات: **P0** (لا تشتغل/تمتثل الشركة بدونه) · **P1** (احترافية وتميّز) · **P2** (تحسينات). تاريخ التدقيق: 2026-07-13.

## ما هو مكتمل فعلًا (الأساس القويّ)
دورة الوساطة الأساسية مبنية باحترافية: العملاء + بوّابة الالتزام · الكتالوج + النموذج الديناميكي · الطلبات → السلِبس/عروض الأسعار → المقارنة → الإسناد · الإصدار + الموافقة الفنية/المالية + الملاحق · التجديدات · المطالبات (+محادثة) · خدمة العملاء (+محادثة) · بوّابة العميل · المحاسبة المزدوجة (Voucher/COA) + فاتورة ZATCA المرحلة 2 + دورة تحصيل/تسوية/ميزان مراجعة · العمولات (وسطاء فرعيون) · الإشعارات + التدقيق + MFA + DLP + الاحتفاظ. **e2e 287/287.** النواقص أدناه تُكمِّل هذا الأساس لا تستبدله. **e2e 325/325** بعد المالية + البوّابة + الدفع + عنقود الامتثال الكامل (الشكاوى + كشف المؤمِّن + AML) + التدفّق النقدي (القوائم المالية الثلاث مكتملة).

## نطاق المرحلة المالية الأولى (مؤكّد) + قرارات تصميمية

تأكيدٌ أن **تقسيط العميل + عمولات الموظفين + سندات صرف العمولات + تفاصيل عمولات الموظفين والوسطاء الفرعيين** ضمن نطاق البناء المالي الأول (بنود 2.1/3.1/3.2 أعلاه). ملاحظات الخبير لضمان **دقّة البيانات**:

- **ما هو موجود فعلًا (نُبرزه لا نبنيه):** عمولات **الوسطاء الفرعيين** ودفترهم **وسند صرفهم** مبنية (`/producers/:id/settle` ⇒ سند PYV بقيد مصروف حساب `05010`). المطلوب: **توحيدها في القسم المالي** ضمن تبويب «العمولات».
- **ما هو جديد بالكامل:** عمولات **الموظفين** (دفتر + تفاصيل لكل وثيقة/فترة + سند صرف PYV بحساب مصروف جديد «عمولات/حوافز الموظفين» يُضاف للـCOA). تصنيفها: **تعويض داخلي (HR/مالي)، لا التزام تنظيمي**.
- **قرار تصميمي — استحقاق عمولة الموظف:** الموصى به **عند تحصيل الوسيط لعمولته من المؤمِّن** (لا عند الإصدار) — أأمن وأأدقّ ماليًا (لا ندفع على عمل قد يسقط)؛ تُعرَض بحالات «متوقّعة/مستحقّة/مدفوعة».
- **قرار تصميمي — تقسيط العميل مقابل تسوية المؤمِّن:** الموصى به أن يكون جدول تقسيط العميل **مستقلًّا** عن توقيت تسوية المؤمِّن (الوسيط يورّد حسب اتفاقه، والتقسيط شأن تحصيلي)؛ الذمّة تُثبَت **كاملة** عند الإصدار وكل قسط = سند قبض. (يُضاف لاحقًا خيار «تقسيط متزامن مع المؤمِّن» عند الحاجة.)
