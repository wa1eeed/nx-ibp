# الموارد البشرية (HR) — وحدة احترافية

وحدة موارد بشرية متكاملة تُغني شركة الوساطة عن برمجيات منفصلة. محكومة بصلاحية موديول `hr` (منفصلة عن `settings`)، ومتاحة لكل الباقات. البيانات الحسّاسة مشفّرة at-rest.

المكوّنات:
- **الرواتب** (`payroll`) — §8.1، موجودة: كشف شهري + ترحيل لسند مصروف.
- **الإجازات** (`leave`) — §8.2، موجودة: طلب الموظف + بتّ الإدارة.
- **الهيكل التنظيمي** (`org`) — موجود: أقسام + أدوار افتراضية.
- **ملفّات الموظفين + الوثائق** — *(هذا الملف، القسم 1)*.
- **الحضور والانصراف** — *(القسم 2)*.
- **التوظيف/الإنهاء (Onboarding/Offboarding)** — *(القسم 3)*؛ فعل الإنهاء نفسه في [38 §4](./38-super-admin-console.md).

## 1. ملفّ الموظف الوظيفي + الوثائق

على تبويب «الملف الوظيفي» في `/tenant/settings/staff/[id]` — عبر وحدة [`hr`](../apps/api/src/modules/hr/hr.service.ts) (`@Authorize({ module: "hr" })`):

### بيانات التوظيف — `GET/PUT /hr/employees/:id/profile`
حقول على `User`: المسمّى الوظيفي · تاريخ التعيين · تاريخ الميلاد · الجنسية · الجوال · رقم الهوية/الإقامة · انتهاء الهوية/الإقامة · الراتب الأساسي · جهة اتصال الطوارئ · العنوان.
- **التشفير at-rest**: `nationalId` و`phone` تُشفَّر بـ**AES-256-GCM** ([`CryptoVaultService`](../apps/api/src/common/crypto/crypto-vault.service.ts)) عند الحفظ، وتُفَكّ عند القراءة **فقط** لمن يملك `hr`. `baseSalary` حسّاس ومحكوم بـ`hr` أيضًا (يُعبّئ كشف الرواتب لاحقًا).
- تحديث جزئي (كل الحقول اختيارية) + تدقيق (`employee_profile`).

### الوثائق — `GET/POST /hr/employees/:id/documents` · `DELETE /hr/documents/:id`
[`EmployeeDocument`](../packages/db/prisma/schema.prisma): عقد/هوية/إقامة/جواز/شهادة/أخرى، برقم وتاريخَي إصدار وانتهاء ورابط اختياري.
- **تنبيه الانتهاء**: `GET /hr/expiring?days=60` يجمع الوثائق **وهويات/إقامات الموظفين** المنتهية/الوشيكة (≤60 يومًا) بأسماء أصحابها — للمتابعة الاستباقية. الواجهة تلوّن التواريخ (أحمر للمنتهي، كهرماني ≤60 يومًا).
- **التخصيص**: تبويب الملف الوظيفي يظهر لمن يملك `hr` (بلا صلاحية ⇒ رسالة لطيفة، والـbackend يفرض 403).

### الاختبار
[`hr.e2e-spec.ts`](../apps/api/test/hr.e2e-spec.ts): تشفير الهوية/الجوال at-rest (مخزَّن مشفّرًا، مُعاد مفكوكًا) · دورة حياة الوثيقة + الوشيك على الانتهاء · عزل صلاحية `hr` (403). **3/3**.

## 2. الحضور والانصراف (تصميم ذكي)

سجلّ حضور يومي واحد لكل موظف ([`Attendance`](../packages/db/prisma/schema.prisma))، بتصميم يوازن بين الأتمتة والدقّة:
- **حضور تلقائي عند تسجيل الدخول**: [`AuthService.login`](../apps/api/src/modules/auth/auth.service.ts) يستدعي `HrService.recordLogin` (غير حاجب) فيُسجَّل حضور اليوم بمصدر `login` إن لم يكن مسجَّلًا — صفر احتكاك، لا يُنسى.
- **تسجيل يدوي دقيق** (Check-in/Check-out): أداة على لوحة التحكّم [`AttendanceWidget`](../apps/web/src/components/hr/AttendanceWidget.tsx) — زرّ حضور/انصراف + وقت اليوم + «اكتمل يومك». `POST /hr/attendance/check-in` · `/check-out`؛ `GET /hr/attendance/today`.
- **سجلّي الشخصي**: `GET /hr/attendance/mine?days=30` — تاريخ + حضور + انصراف + المدة (لكل موظف؛ خدمة ذاتية).
- **لوحة حضور الفريق** (مديرون، `hr`): `GET /hr/attendance/team?date=` — لكل موظف نشِط حالته اليوم: **حاضر / انصرف / غائب** + بحث بالتاريخ. صفحة `/tenant/settings/attendance`.
- **مفتاح اليوم** موحّد (منتصف ليل UTC ليوم الرياض) لمطابقة `@db.Date` بلا التباس منطقة زمنية.
- **الاختبار**: الحضور التلقائي عند الدخول (مصدر login) · الانصراف اليدوي · سجلّي · لوحة الفريق — ضمن `hr.e2e-spec.ts` (**6/6**). مُتحقَّق حيًّا.

## 3. قوائم التعيين/الإنهاء (Onboarding/Offboarding)

قوائم متابعة إجراءات الالتحاق والمغادرة ([`EmployeeChecklistItem`](../packages/db/prisma/schema.prisma)) — على تبويب «الملف الوظيفي»:
- **بذر تلقائي**: عند **إنشاء موظف** ([`StaffService.create`](../apps/api/src/modules/staff/staff.service.ts)) تُبذَر بنود التعيين الافتراضية (إنشاء الحساب [مكتمل] · توقيع العقد · تسليم العهدة · بيانات الهوية · التعريف بالفريق). وعند **إنهاء الخدمة** ([`offboard`](../apps/api/src/modules/staff/staff.service.ts)) تُبذَر بنود المغادرة (استرجاع العهدة · تعطيل الوصول · تسوية المستحقّات · تسليم المهام · المخالصة).
- **الإدارة**: `GET /hr/employees/:id/checklist` · `POST /hr/checklist/:id/toggle` · `POST /hr/employees/:id/checklist` (إضافة) · `DELETE /hr/checklist/:id` — كلها `hr`.
- **الواجهة**: مربّعات إنجاز بشريط تقدّم ونسبة (done/total) + إضافة/حذف بنود مخصّصة.
- **الاختبار**: البذر عند الإنشاء (البند الأول مكتمل) + التبديل + الإضافة/الحذف — `hr.e2e-spec.ts` **7/7**.
