# متطلبات وساطة التأمين — مصفوفة الأمان ودورة العمل (مرجع متتبَّع)

> مصدره مواصفة متطلبات العميل (وثيقة وظيفية مرجعية). يُقرأ مع `BLUEPRINT.md` و`ROADMAP.md`.
> **ملاحظة تنظيمية:** الإشراف على التأمين انتقل إلى **هيئة التأمين (Insurance Authority)** المنشأة 2023، وتولّت مهام كانت لدى مؤسسة النقد (ساما). لذا التقارير الرقابية تُوجَّه إلى **هيئة التأمين**، والفوترة الإلكترونية إلى **ZATCA**.

## 1) مصفوفة الأمان — الأدوار السبعة الأساسية

| # | الدور (الوثيقة) | الصلاحيات الحسّاسة | مقابلها لدينا (`packages/shared/src/rbac.ts`) |
|---|---|---|---|
| 1 | Super Admin / الإدارة العليا | كامل الحوكمة، توزيع الأدوار، التقارير العليا عبر الفروع/المبيعات/مراكز التكلفة | `general_manager` (داخل المستأجر) + سوبر أدمن المنصة (المرحلة 8) |
| 2 | Salesman / تطوير الأعمال | الفرص البيعية (Leads)، KYC ونماذج الطلب والمرفقات، تقارير مبيعاته + Win/Loss | `sales_rep` + `sales_manager` |
| 3 | Compliance Officer | مراجعة العملاء المحوّلين، **قبول/رفض قانوني قبل طلب الأسعار** | `compliance_manager` (+ بوّابة سير العمل: المرحلة 3) |
| 4 | Underwriter / الاكتتاب | طلبات الأسعار (Slips/RFQ)، جدول المقارنات، إدخال الوثيقة/الملحق، **الموافقة الفنية**، الأرشفة والتسليم | `pricing_officer` + `policy_admin` (+ سير العمل: المراحل 3–4) |
| 5 | Customer Service | استقبال وتدقيق الطلبات اليومية، المراسلات التلقائية، تقارير الكفاءة | `customer_care_manager` (الموديول: المرحلة 6) |
| 6 | Claims Officer | تسجيل المطالبات، متابعة التسوية والتحمّل، الإغلاق والفوترة | `claims_officer` (الموديول: المرحلة 6) |
| 7 | Accountant / المالية | اعتماد الحساب في COA (داخل/خارج الميزانية)، **الاعتماد المالي** (ترحيل القيود + إشعارات مدين/دائن + فواتير ضريبية)، السندات (RCV/PYV/JRV/DPV)، التقارير المالية والرقابية | `accountant` (الموديول: المرحلة 4) |

> تحكّمنا أدقّ من «دور واحد مطلوب»: فحص **entitlement الباقة + مصفوفة CRUD لكل موديول قابلة للتعديل لكل موظف**.

## 2) المتطلبات الفنية والأمنية

- **هيكلية موديولات معزولة:** module لكل مجال (نُفّذت بـ NestJS modules بدل مثال الـ Python).
- **عزل المستأجرين:** استخراج `tenant_id` من التوكن وحقنه في سياق الطلب. *(لدينا: ALS + Prisma middleware يفرض الفلترة + FK على مستوى DB.)*
- **RBAC middleware:** فحص الأدوار/الصلاحيات قبل الوصول لأي موديول.
- **مدير التخزين:** ضغط الصور إلى WebP، حد حجم الملف، مسار `tenant_{id}/<module>/`، **روابط موقّتة 5 دقائق (Presigned URLs)**، تشفير، فحص MIME. *(حد الرفع = entitlement لكل باقة يضبطه السوبر أدمن.)*
- **مولّد الأرقام التسلسلية:** `PREFIX-BRANCH-CLASS-YEAR-SEQ` (مثل `POL-RUH-MED-2026-1001`) يبدأ 1001 لكل (مستأجر/فرع/فئة/سنة).
- **حياد سحابي:** `.env` يوجّه القاعدة والتخزين (`STORAGE_DRIVER`) للبيئة الحالية (Coolify VPS → AWS/GCP/Alibaba) دون لمس الكود.

## 3) الامتثال التنظيمي (هيئة التأمين + PDPL + NCA)

- **فصل أموال العملاء (Fiduciary):** شجرة حسابات بحسابات **داخل الميزانية (On‑Balance)** و**خارجها (Off‑Balance)** — الطريقة القانونية لفصل أقساط العملاء عن حسابات الوسيط.
- **سلسلة التدقيق (Audit Trail):** تتبّع كل وثيقة/حركة من الفرصة البيعية حتى التسوية، بأرقام تسلسل غير قابلة للتلاعب.
- **PDPL/NCA:** لا روابط عامة، روابط موقّتة 5 دقائق، عزل منطقي بالـ Object Storage، تشفير، وتوطين بيانات الإنتاج (والنسخ/السجلات/المرفقات) داخل المملكة.
