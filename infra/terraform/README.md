# infra/terraform — البنية التحتية ككود

هيكل **حياديّ سحابياً** (AWS/GCP/Alibaba) يفرض القواعد غير القابلة للتفاوض.

| الملف | المحتوى |
|---|---|
| `variables.tf` | المزوّد، المنطقة (داخل المملكة)، البيئة، أحجام الموارد، حارس التوطين |
| `main.tf` | حارس التوطين (`precondition` يمنع الإنتاج خارج المملكة) + خريطة الوحدات |
| `outputs.tf` | مخرجات (مراجع أسرار لا قيم) |

**القواعد المفروضة (CLAUDE.md §3):**
- **التوطين:** بيئة الإنتاج محصورة في مناطق داخل المملكة (AWS `me-central-1`، GCP `me-central2`، Alibaba `me-central-1`) — يفشل `terraform plan` خارجها.
- **التشفير:** at-rest (قاعدة/تخزين/حالة) و in-transit (TLS) في كل الوحدات.
- **الأسرار:** من مدير أسرار المزوّد فقط — لا قيم في الكود ولا في الحالة العلنية.
- **النسخ الاحتياطية والـ logs والمرفقات:** موطّنة داخل المملكة.

**الوحدات (تُجسَّد لكل مزوّد):** `network` · `k8s` · `database` (PostgreSQL مُدار) · `cache` (Redis) · `storage` (كائنات) · `secrets` · `logging`.

> أبقِ الحالة (state) والأسرار خارج المستودع. التكامل الحكومي (ZATCA/نفاذ/...) عبر نقاط الإنتاج فقط في هذه البيئة؛ التطوير عبر Sandbox.
